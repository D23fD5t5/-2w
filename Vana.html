Суть проблемы и цель решения кейса

Текст кейса: В компании используются простые пароли. Нужно разработать сбалансированную политику — безопасную, но удобную для пользователей.

Наша цель: Создать такие правила для паролей, которые будут надежно защищать данные, но при этом сотрудники не будут их записывать на бумажки или записывать в избранное в соц. сетях.

Целевая аудитория

Для кого актуальна ситуация?

  • Все сотрудники компании, которые используют учетные записи для доступа к корпоративным системам.
  • Специалисты по информационной безопасности и IT-администраторы, отвечающие за настройку политик безопасности.

Кому востребовано наше решение?

  • Руководству компании — для снижения рисков кибератак.
  • Сотрудникам — для понятных и выполнимых правил безопасности без лишнего стресса.
  • HR и отделу обучения — для внедрения инструкций и обучения персонала.

Основные правила создания паролей

Минимальная длина

12 символов

Комбинация символов

Буквы разного регистра, цифры, специальные символы (не обязательно в каждом пароле)

Запрещено использовать

Личные данные (имена, даты рождения), общеизвестные слова и фразы, последовательности символов (Dima, 123456)

Периодичность смены

Сбалансированный подход без чрезмерно частой смены

Многофакторная аутентификация

Обязательна для критически важных систем

Решение: «Политика сбалансированных паролей»

Длина важнее сложности

Пароль должен быть не менее 12-15 символов.

Используйте запоминающуюся фразу

Например, «snowy_winter2025(Снежная зима)» вместо «winter123».

Проверка на утечки

Использование сервисов для проверки, не был ли пароль раскрыт в публичных утечках данных.

Почему это оптимально?

Преимущества

  • Длинные фразы-пароли устойчивы к взлому
  • Их легче запомнить
  • Отказ от частой смены снимает нагрузку с пользователя
  • Уменьшается желание записать пароль

Альтернатива «стикерам»: Менеджеры паролей

Это специальные программы, которые надежно хранят все ваши пароли под одним главным — «мастер-паролем».

Решение позволяет использовать уникальные и сложные пароли для каждого сервиса, не запоминая их все.

Примеры:

Bitwarden

Открытое программное обеспечение с бесплатным тарифом

KeePass

Локальный менеджер паролей с открытым исходным кодом

Вывод

Мы проанализировали проблему использования простых паролей и выяснили, что часто виной тому являются неудобные и устаревшие политики безопасности.

Наше решение смещает фокус с создания сложных для запоминания паролей на создание длинных и удобных парольных фраз.

Внедрение предложенных правил в сочетании с рекомендацией использовать менеджеры паролей позволит значительно повысить уровень безопасности в компании, сделав его при этом комфортным для сотрудников.
